# GitHub Pages 空白页面排查清单

## ✅ 已确认正确的配置
- [x] `vite.config.js` 中 `base: '/ENTwebsite/'` 已配置
- [x] `.github/workflows/deploy.yml` 部署脚本正确
- [x] `package.json` 包含 `build` 脚本
- [x] React 应用代码正常

## 🔍 需要检查的项目

### 1. GitHub Pages 设置
1. 访问：https://github.com/Broderick-23/ENTwebsite/settings/pages
2. 确认 **Source** 设置为：`GitHub Actions` 
   - ❌ 如果是 "Deploy from a branch"，需要改为 "GitHub Actions"

### 2. GitHub Actions 运行状态
1. 访问：https://github.com/Broderick-23/ENTwebsite/actions
2. 查看最近的工作流运行记录：
   - ✅ 绿色勾号 = 部署成功
   - ❌ 红色叉号 = 部署失败（点击查看日志）
   - ⚪ 灰色圆点 = 正在运行中

### 3. 访问地址确认
正确的访问地址应该是：
```
https://broderick-23.github.io/ENTwebsite/
```
❌ 错误：https://broderick-23.github.io/（会显示 404 或其他内容）

## 🚀 手动触发部署（如果 Actions 未运行）

### 方法一：通过 Git 命令
```bash
# 在项目根目录执行
git add .
git commit -m "Trigger deployment" --allow-empty
git push origin main
```

### 方法二：在 GitHub 网页手动触发
1. 访问：https://github.com/Broderick-23/ENTwebsite/actions
2. 点击左侧 "Deploy to GitHub Pages"
3. 点击右侧 "Run workflow" → "Run workflow"

## 🔧 如果仍然空白，检查浏览器控制台

1. 打开部署的网页
2. 按 `F12` 打开开发者工具
3. 查看 `Console` 标签页是否有错误
4. 查看 `Network` 标签页，刷新页面，检查：
   - `index.html` 是否加载成功（200 状态）
   - JS 文件（如 `index-xxx.js`）是否加载成功
   - 如果 JS 文件返回 404，说明路径配置有问题

## 📋 常见错误及解决方案

### 错误1：JS 文件 404
**原因**：base 路径配置错误  
**解决**：确认 `vite.config.js` 中 `base: '/ENTwebsite/'` 且仓库名确实是 `ENTwebsite`

### 错误2：页面显示 404
**原因**：访问地址错误  
**解决**：确保访问 `https://broderick-23.github.io/ENTwebsite/`（末尾带仓库名）

### 错误3：Actions 运行失败
**原因**：依赖安装失败或构建错误  
**解决**：查看 Actions 日志，根据具体错误修复

## 🎯 快速验证

在本地运行以下命令验证构建是否成功：
```bash
npm run build
```
检查生成的 `dist/index.html` 文件，应该包含类似这样的脚本标签：
```html
<script type="module" crossorigin src="/ENTwebsite/assets/index-xxxxx.js"></script>
```
注意路径中应该包含 `/ENTwebsite/` 前缀！
