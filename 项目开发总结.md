# MT App - 心理健康伙伴项目开发总结

## 📋 项目概述

### 项目名称
MT App Landing Page - 专为中国大学生打造的心理健康伙伴

### 项目目标
- **用户群体**: 中国大学生
- **核心使命**: 利用先进AI技术对心理困扰进行正向干预
- **品牌理念**: "Thoughts Heard, Mind Nurtured" (倾听心声，抚慰心灵)

### 功能特色
1. **即时心情记录** - 轻松记录情绪，生成个性化情绪日记

## 🐛 技术问题与解决历程

### 问题一: GitHub Pages 显示空白页面

#### 🔍 问题现象
- GitHub Pages 部署成功（绿色勾号）
- 网页能显示标题和图标
- 页面内容完全空白

#### 🕵️ 问题分析
通过排查发现两个关键问题：

1. **缺少 `postcss.config.js` 文件**
   - Tailwind CSS 无法正常编译
   - 导致所有样式失效，页面"排版混乱，没有色块"

2. **误删除了关键脚本标签**
   - `index.html` 中的 `<script type="module" src="/src/main.jsx"></script>` 被删除
   - React 应用根本无法启动

#### ✅ 解决方案

**步骤一: 创建 PostCSS 配置**
```javascript
// postcss.config.js
export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

**步骤二: 恢复脚本标签**
```html
<!-- index.html -->
<script type="module" src="/src/main.jsx"></script>
```

**步骤三: 修正CSS导入顺序**
```css
/* src/index.css - @import 必须在 @tailwind 之前 */
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@100..900&display=swap');

@tailwind base;
@tailwind components;
@tailwind utilities;
```

**步骤四: 优化Git配置**
```
# .gitignore
node_modules/
dist/
package-lock.json
```

### 问题二: 构建配置优化

#### 🔍 Vite配置要点
```javascript
// vite.config.js
export default defineConfig({
  base: '/ENTwebsite/',  // GitHub Pages 路径配置
  plugins: [react()],
  build: {
    outDir: 'dist',      // 输出目录
  }
})
```

#### 🔍 GitHub Actions 部署配置
```yaml
# .github/workflows/deploy.yml
- name: Build Project with Vite
  run: npm run build

- name: Deploy to GitHub Pages
  uses: actions/deploy-pages@v4
  with:
    artifact_path: './dist'  # 部署构建后的文件
```

## 💡 经验教训

### 关键学习
1. **配置文件的完整性至关重要** - PostCSS 配置的缺失导致了样式编译失败
2. **脚本加载的正确性** - 错误删除脚本标签会导致应用无法启动
3. **系统性调试方法** - 遇到问题时要从构建流程、文件配置、网络请求等多角度排查
4. **Git工作流的重要性** - 正确的 .gitignore 配置能避免不必要的文件提交

### 最佳实践
- 始终保持配置文件的完整性
- 定期验证构建和部署流程
- 建立系统性的问题排查方法
- 重视用户体验的每一个细节
